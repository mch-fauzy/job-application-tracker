# Job Application Tracker — Plan 1: Foundation & Shared Infrastructure

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the Next.js + Hono + Drizzle + Neon project skeleton and the `shared/` infrastructure layer (config, db, base columns, the immutable `audit_log` + `recordAudit`, the response envelope, message constants, and the TanStack query client) that every feature depends on.

**Architecture:** One unified Next.js (App Router) app, Feature-Based "Pattern B". This plan builds only `shared/` + project setup — no feature code. The cross-cutting `audit_log` schema and `recordAudit` writer live in `shared/` so any feature service can record an audit row without a cross-feature import. The audit log is immutable at the DB level (a `BEFORE UPDATE OR DELETE` trigger that `RAISE`s, plus `REVOKE`).

**Tech Stack:** Next.js, React, TypeScript, Hono, Drizzle ORM (`drizzle-orm/neon-serverless`), `@neondatabase/serverless`, `ws`, drizzle-kit, Zod, `@tanstack/react-query`, Vitest, ESLint (`eslint-plugin-boundaries`, `eslint-plugin-project-structure`), knip, husky. npm.

## Global Constraints

Every task implicitly includes these (copied from the spec + rules):

- **camelCase-only on the wire** (input AND output). Drizzle maps to snake_case DB columns. No snake_case middleware.
- **Strings use `text`**, never `varchar`. Length validated in Zod, not the column.
- **Status set is `text` + a Zod enum — NEVER `pgEnum`, no DB CHECK.** (Plan 2.)
- **Keyset/cursor pagination only**, ordered `(updatedAt, id)`. Never `OFFSET`.
- **Soft delete** via `deletedAt`; queries filter `deletedAt IS NULL`.
- **`server-only` boundary:** every file that touches the DB or server logic begins with `import 'server-only'`. A `'use client'` file importing one is a build error.
- **No cross-feature imports.** `shared/` is importable by anything; features import only `shared/` + their own files.
- **Folder-based naming, no type suffix, no `I` prefix.** API versioned by folder (`api/v1/`, `dtos/v1/`).
- **Audit log is immutable + append-only**, written in the same transaction as each mutation.
- **TDD, ≥80% coverage.** Hono route handler runs on the **Node runtime** (`neon-serverless` needs it).
- **Env vars:** `DATABASE_URL` (pooled `-pooler`, app), `DATABASE_URL_UNPOOLED` (unpooled, migrations + tests).
- **Git is run by the user.** The executing agent NEVER runs `git add/commit/push`. When all of this plan's tasks are green, pause and surface the suggested commit command (see the Commit section at the end) for the user to run. One commit per plan.

**Execution prerequisite:** a Neon database. Provide `DATABASE_URL` (pooled) + `DATABASE_URL_UNPOOLED` (unpooled) in `.env` before Task 8. A dedicated Neon **test branch** is recommended so tests never touch real data.

```ts
function required(env: Record<string, string | undefined>, key: string): string {
```

**Files:**

**Interfaces:**
- Produces: a `BEFORE UPDATE OR DELETE` trigger on `audit_log` that raises, plus `REVOKE UPDATE, DELETE` (defense-in-depth).

- [ ] **Step 1: Generate an empty custom migration**

Expected: a new empty `.sql` file under `src/shared/db/migrations/`.

- [ ] **Step 2: Hand-write the SQL into that file**

```sql
-- audit_log is append-only: block UPDATE and DELETE
CREATE OR REPLACE FUNCTION audit_log_block_mutation() RETURNS trigger AS $$
BEGIN
  RAISE EXCEPTION 'audit_log is append-only: % is not allowed', TG_OP;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER audit_log_immutable
  BEFORE UPDATE OR DELETE ON audit_log
  FOR EACH ROW EXECUTE FUNCTION audit_log_block_mutation();

-- defense-in-depth: fully effective when the app connects as a non-owner role
REVOKE UPDATE, DELETE ON audit_log FROM PUBLIC;
```

```ts
// src/shared/test/db.ts
import { db } from '@/shared/lib/db';
import type { DbTransaction } from '@/shared/lib/db';

class Rollback extends Error {}

export async function withRollback(fn: (tx: DbTransaction) => Promise<void>): Promise<void> {
  try {
    await db.transaction(async (tx) => {
      await fn(tx);
      throw new Rollback();
    });
  } catch (e) {
    if (!(e instanceof Rollback)) throw e;
  }
}
```

```ts
// src/shared/lib/query-client.ts
import { QueryClient, MutationCache, matchQuery, isServer, type QueryKey } from '@tanstack/react-query';

declare module '@tanstack/react-query' {
  interface Register { mutationMeta: { invalidates?: QueryKey[] } }
}

function makeQueryClient() {
  const client = new QueryClient({
    defaultOptions: { queries: { staleTime: 60_000 } },
    mutationCache: new MutationCache({
      onSuccess: (_data, _vars, _ctx, mutation) => {
        client.invalidateQueries({
          predicate: (query) =>
            mutation.meta?.invalidates?.some((key) => matchQuery({ queryKey: key }, query)) ?? true,
        });
      },
    }),
  });
  return client;
}

let browserClient: QueryClient | undefined;
export function getQueryClient() {
  if (isServer) return makeQueryClient();   // fresh per request (RSC prefetch)
  return (browserClient ??= makeQueryClient()); // singleton in the browser
}
```
